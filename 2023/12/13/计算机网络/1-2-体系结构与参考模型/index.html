1.2 体系结构与参考模型

Ableson Lv2
  2023-12-13 21:21:13 2023-12-13 21:21:13 创建   2023-12-14 14:09:46 2023-12-14 14:09:46 更新  
  • 计算机网络 
 

计算机网络的体系结构是指层次、层次的协议以及层间的接口的集合；

分层结构中的概念

计算机网络分层，各个层之间相对独立，有助于更新、调试，灵活性好；但是层次越多，可能有些功能在不同层重复出现，产生额外开销，降低运行效率；层次越少，每个层的任务越多，协议越复杂；

  • 实体：实体是指使用硬件或软件对某一层功能的实现，第 n 层的活动元素称为 n 层实体，同一层的实体叫对等实体；

  • 协议：为了网络中对等实体之间进行数据交换而建立的规则、标准或约定，称为网络协议；

    • 语法：规定传输数据的格式，规定比特序列几位分割、多少位是一帧等；
    • 语义：规定所要完成的功能，规定比特序列各个位的含义；
    • 同步：规定各种操作的顺序，

  • 接口：上层使用下层服务的入口，同一结点的相邻两层实体通过服务访问点 SAP 进行交互；

  • 服务：下层为上层提供的功能调用；上层使用下层提供的服务时使用的命令叫做服务原语，OSI 参考模型将服务原语分为四类，这四类原语可以用于建立连接等；

    • 请求：服务用户发给服务提供者，表示需要使用请求功能；
    • 证实：服务提供者发给服务用户，表示对请求的响应；
    • 响应：服务用户发给服务提供者，表示对指示的响应；
    • 指示：服务提供者发给服务用户，告诉用户有某件事情将要发生；

服务的本质是下一层中可以被上一层使用的功能，只有能被上层看到的功能才叫服务；

上一层可以利用下一层的四种功能，也就是四类服务原语；

计算机网络提供的服务

  • 面向连接服务与无连接服务

    • 面向连接服务：通信前双方先建立连接，预分配相应的资源，以保证通信能够正常进行，相当于形成了一个逻辑上的连接；传输结束后释放连接和占用的资源；
    • 无连接服务：需要发送数据不需要提前通知，直接开始发送数据；一次发送的不同数据块可以有不同的传输路径；

  • 可靠服务和不可靠服务

    • 可靠服务：网络具有纠错、检错、应答机制，通过确认和重传机制保证接收方正确接收到信息；
    • 不可靠服务：普通传输，当接收方发现有错误时，仅仅丢弃错误信息；

  • 有应答服务和无应答服务

  • 无确认无连接服务：不预先建立链路连接，目的机器收到数据以后不发回确认，丢失的帧，本层不负责重发而交给上层处理；适用于实时通信或者误码率较低的信道，如以太网；

  • 有确认无连接服务：不预先建立链路连接，但是目的机器收到数据帧时需要发回确认，源机器在规定时间内没有收到确认，就重发数据帧，以提高数据传输的可靠性；适用于误码率较高的数据通信，如无线网；

  • 有确认面向连接服务：帧传输分为三个阶段，建立数据链路、传输数据帧、释放数据链路，目的机器对每一帧都需要进行确认，然后才进行下一帧的传输，适用于通信质量要求较高的场合；

协议中数据单元的组成

  • SDU：服务数据单元，为了完成用户所要求的功能，而传送的数据，协议栈上下层之间传输的数据；
  • PCI：协议控制信息，控制协议操作的附加信息；
  • PDU：协议数据单元，对等层次之间传送的数据单位；

上一层的 PDU 作为下一层的 SDU；

最大传输单元MTU

MTU是指上层传输给本层的最大的数据的最大长度，是本层数据部分SDU的最大长度，而不是本层PDU的最大长度；

分层结构的参考模型

上两层端到端通信，看不到底层传输的细节，数据直接从起点到终点，下三层是点到点传输，每一次只关心数据下一步向哪里走；

各个层的功能

TCP/IP层级 OSI层级 协议 数据单元 功能
应用层 应用层 域名系统DNS、HTTP协议、SMTP协议等 报文 服务用户；
表示层 信息表示方式、数据格式变换、
会话层 建立同步
传输层 传输层 TCP协议：面向连接的、可靠的数据传输服务 报文段 实现两个主机端到端的通信；可靠传输、差错控制、流量控制、复用和分用；
UDP协议：无连接的数据传输服务 用户数据报

网路接口层 数据链路层 SDLC、HDLC、PPP、STP、帧中继等 帧 透明传输；封装成帧；差错控制、流量控制、传输管理等；

注意

  • 数据传输使用的双绞线等物理媒介并不属于物理层的内容，物理层只涉及统一的规则；

OSI模型和TCP/IP模型的区别

  • 传输层和网络层的功能划分不同

    • OSI模型规定网络层实现无连接和面向连接的服务，传输层只实现面向连接的服务；
    • TCP/IP协议中，网络层的IP 协议是无连接的，因此网际层提供的是无连接的服务，TCP 和 UDP 共同在传输层提供了面向连接和无连接的服务；

  • OSI模型中会话层和表示层设置较为繁琐，在TCP/IP模型实现中，直接融合到了应用层中；

  • TCP/IP模型没有具体的数据链路层和物理层的划分，只有网路接口层；
